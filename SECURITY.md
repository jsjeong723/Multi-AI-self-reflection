# Security Policy (보안 정책)

## 🧭 프로젝트의 보안 철학
`Multi-AI-self-reflection` 프로젝트는 고등학생의 성찰을 돕는 교육 플랫폼 아키텍처입니다. 특히 **Secret 세션(1:1 AI 채팅)**의 학생 프라이버시 보호와 **위기 대응 프로토콜**의 안정성은 본 프로젝트의 최우선 가치입니다. 

우리는 기술적 취약점이나 데이터 누출 리스크가 학생과 교실 환경에 치명적인 영향을 미칠 수 있음을 인지하고 있으며, 발견된 취약점에 대해 신속하고 안전하게 대응하고자 합니다.

## 🚨 취약점 제보 방법 (Reporting a Vulnerability)

본 프로젝트의 소스코드, 프롬프트 아키텍처, 혹은 데이터 흐름에서 보안 취약점(예: Secret 세션 데이터 유출 위험, 위기 감지 오작동 등)을 발견하신 경우, **절대로 GitHub Issue에 공개적으로 등록하지 마십시오.** 취약점이 악용되는 것을 막기 위해 아래의 프로세스를 따라 비밀리에 제보해 주시기 바랍니다.

1. **비공개 제보 채널:** 아래의 이메일 주소로 취약점 내용을 보내주세요.
   * 📧 **Contact Email:** `jjsool723@gmail.com` *
2. **포함할 내용:**
   * 발견된 취약점의 종류 및 위치 (파일명, 라인 번호 또는 매커니즘 구조)
   * 취약점을 재현할 수 있는 방법 또는 시나리오 설명
   * 이 취약점이 실제 학교 런타임(교실) 환경에 미칠 수 있는 영향력 평가
3. **확인 및 회신:** 
   * 제보가 접수되면 **48시간 이내**에 확인 메일을 발송하며, 취약점 분석 및 패치 타임라인을 공유합니다.

---

## 🛠️ 취약점 대응 절차 (Our Response Process)

1. **접수 및 격리 분석:** 제보된 취약점을 내부 비공개 환경에서 재현하고 영향을 분석합니다.
2. **패치 개발:** 메인(main) 브랜치에 직접 커밋하지 않고, 비공개 보안 브랜치에서 패치 코드를 작성 및 테스트합니다.
3. **버전 업데이트:** 패치가 완료되면 `v1.2.1` 또는 `v1.3`과 같이 마이너 버전을 릴리즈하며 변경 로그(Changelog)에 보안 패치 사실을 명시합니다. (제보자가 원할 경우 기여자 목록에 성함을 등재합니다.)
4. **공개 고지:** 패치가 배포된 후, 필요한 경우 리포지토리 메인 페이지를 통해 해당 취약점의 해결 순서를 커뮤니티에 공유합니다.

---

## ⚖️ 기밀 유지 협약 (Responsible Disclosure)

우리는 오픈소스 생태계의 발전을 신뢰합니다. 발견된 취약점이 완전히 해결되어 안전한 패치 버전이 배포될 때까지, 취약점의 세부 내용을 대중이나 미디어에 공개하지 말아 주시기를 정중히 부탁드립니다. 학생들의 안전한 성찰 공간을 지키기 위한 커뮤니티의 협조에 깊이 감사드립니다.
